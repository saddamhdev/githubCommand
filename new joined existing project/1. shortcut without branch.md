Good question — this is where many beginners get confused 👍

## 🔹 Short Answer

👉 **After first push (with `-u`) → just use:**

```bash
git push
```

---

## 🔹 Why?

### First time (you already did this):

```bash
git push -u origin feature/saddamnvn
```

👉 This sets **upstream (tracking branch)**

---

### After that (daily use):

```bash
git push
```

👉 Git already knows:

* remote = `origin`
* branch = `feature/saddamnvn`

---

## 🔹 If you DID NOT use `-u` before

Then you must use:

```bash
git push origin feature/saddamnvn
```

---

## 🔹 Check if upstream is set

```bash
git branch -vv
```

If you see:

```bash
feature/saddamnvn  abc123 [origin/feature/saddamnvn]
```

✅ Then you can use just `git push`

---

## 🔹 Best Practice (for you 🚀)

Always do this once:

```bash
git push -u origin feature/saddamnvn
```

Then forever:

```bash
git push
git pull
```

---

## 🔥 Pro Tip (Very Important)

Same rule applies for pull:

```bash
git pull
```

(no need to write branch again)
